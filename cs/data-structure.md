## 자료구조 개요

> 자주 사용되는 자료구조의 종류와 특징을 설명해주세요.

<details>
<summary>답변 보기</summary>

| 자료구조 | 특징 | 시간복잡도 (검색) |
|---------|------|-----------------|
| 배열(Array) | 인덱스로 빠른 접근, 삽입/삭제 느림 | O(1) |
| 연결 리스트(Linked List) | 삽입/삭제 빠름, 탐색 느림 | O(n) |
| 스택(Stack) | LIFO, 후입선출 | O(n) |
| 큐(Queue) | FIFO, 선입선출 | O(n) |
| 해시 테이블(Hash Table) | 키-값 쌍, 빠른 검색 | O(1) 평균 |
| 트리(Tree) | 계층적 구조, BST는 O(log n) 검색 | O(log n) |
| 그래프(Graph) | 노드와 엣지, BFS/DFS 탐색 | O(V+E) |

</details>

## 스택과 큐의 차이

> 스택(Stack)과 큐(Queue)의 차이점과 각각의 사용 사례를 설명해주세요.

<details>
<summary>답변 보기</summary>

| 구분 | 스택(Stack) | 큐(Queue) |
|------|------------|----------|
| 원리 | LIFO (Last In, First Out) | FIFO (First In, First Out) |
| 삽입 | push (위에 쌓음) | enqueue (뒤에 추가) |
| 삭제 | pop (위에서 꺼냄) | dequeue (앞에서 꺼냄) |
| 사용 사례 | 함수 호출 스택, 브라우저 방문기록, undo/redo | 이벤트 루프, 프린터 대기열, BFS |

**JavaScript에서의 이벤트 루프:**
- **Call Stack** (스택): 실행 중인 함수를 관리
- **Task Queue / Microtask Queue** (큐): 비동기 콜백 대기

</details>

## 프로세스와 스레드의 차이점

> 프로세스(Process)와 스레드(Thread)의 차이점을 설명해주세요.

<details>
<summary>답변 보기</summary>

| 구분 | 프로세스 | 스레드 |
|------|---------|--------|
| 정의 | 실행 중인 프로그램의 인스턴스 | 프로세스 내의 실행 단위 |
| 메모리 | 독립적인 메모리 공간 | 프로세스 메모리 공유 |
| 통신 | IPC(Inter-Process Communication) 필요 | 메모리 공유로 쉽게 통신 |
| 생성 비용 | 높음 | 낮음 |
| 안전성 | 프로세스 간 격리 → 하나 죽어도 다른 프로세스 영향 없음 | 스레드 하나 오류 시 전체 프로세스에 영향 |

**JavaScript는 싱글 스레드:**
- 메인 스레드 하나에서 JavaScript 실행
- 비동기 처리는 이벤트 루프와 Web API로 처리

</details>
